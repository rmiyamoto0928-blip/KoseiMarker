# 校正マーカー (KoseiMarker)

Premiere Pro 用の CEP 拡張パネル。タイムコード付きの校正リストを貼り付けて、シーケンスにマーカーを一括で打てます。校正の指摘を「その位置に飛べるメモ」としてタイムラインに残すためのツールです。

## できること

- タイムコード付きの「現在 / 修正案」リストを貼り付け → シーケンスマーカーを一括生成（コメントに `現在：… / 修正：…`）
- **校正マーカーは赤で打たれ、「✅ 済」で緑に変わる**（済み＝名前が `校正✅` で始まる。色APIが無い環境でも✅名で管理が成立）
- **指示書読込ボタン**… first-cut-review の添削指示書（.md）や誤字脱字チェック結果（.md/.txt）を直接読み込み、修正表だけを抽出して貼り付け欄へ展開
- マーカー一覧（**常に時間順**）をパネル内に表示し、**クリックでその位置へジャンプ**（選択行はハイライト）
- 件数表示…「校正 N件（✅済 n／残 m）」で消し込み進捗が一目でわかる
- 一覧を「全部 / 校正 / 残り / その他」でフィルタ＋**文字検索**
- マーカーの**個別削除（2タップ確認）・コメントのその場編集**
- フレームレート自動判定（`seq.timebase` から実 fps を取得）＋手動選択
- 秒精度TC用の微調整プリセット（30/29.97 系 → 10 フレーム、59.94/60 系 → 30 フレーム）
- 貼り付けデータ／校正マーカーのワンクリック削除（リセット）

## 対応する貼り付け形式

- **縦並び**（1セル1行）… `タイムコード` → `現在` → `修正案` の3行を1組として自動認識
- **タブ区切り**… `タイムコード[Tab]現在[Tab]修正案`
- **Markdown表**… typo-check 統合レポートの3列表（`| TC | 該当テキスト | 修正案 |`）、添削指示書の6列表（`| # | TC | 該当 | 修正案 | 理由 | 重要度 |`）をそのまま貼り付け可
- タイムコードは `HH;MM;SS;FF`（DF）/ `HH:MM:SS:FF` / 短縮形 `MM:SS`・`HH:MM:SS` / SRT形式 `HH:MM:SS,mmm` に対応

## 微調整（フレーム）の考え方

`0:34` のような**秒までしか分からないTC**は秒の頭に刺さり、テロップ表示の中心から少し前にズレる。微調整プリセット（30系=10f／60系=30f）はこれを補正するためのもので、**秒精度のTCにだけ適用**される。`00;00;28;26` や `00:00:10,767` のような**フレーム/ミリ秒精度のTCには適用されず、常に正確な位置**に打たれる。

## インストール

1. このリポジトリを CEP 拡張フォルダに配置：
   ```
   ~/Library/Application Support/Adobe/CEP/extensions/KoseiMarker/   (macOS)
   %APPDATA%\Adobe\CEP\extensions\KoseiMarker\                        (Windows)
   ```
2. 未署名拡張を許可（初回のみ）。macOS の例（CSXS バージョンは環境に合わせる）：
   ```
   defaults write com.adobe.CSXS.11 PlayerDebugMode 1
   defaults write com.adobe.CSXS.12 PlayerDebugMode 1
   ```
3. Premiere Pro を起動 → `ウィンドウ > エクステンション > 校正マーカー`

## 使い方

1. 対象シーケンスを開く（パネル上部にシーケンス名が出る）
2. フレームレートは「自動」でOK
3. 校正リストを貼り付けて「マーカーを追加」
4. 一覧からクリックで各マーカーへ移動

## 注意

- コードを更新したら Premiere を再起動してください（CEP のキャッシュ `~/Library/Caches/CSXS/cep_cache` を削除すると確実）。
- マーカー位置はフレーム番号 × `seq.timebase` で計算しているため、フレーム単位で正確に配置されます。
